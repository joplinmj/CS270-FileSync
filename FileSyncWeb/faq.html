FAQs

Some answers to your frequently asked questions


I am receiving connection timeout errors when running FileSyncClient.jar

There are several possibilities as to why this might happen. All of them stem from not having the correct information in client.properties, server.properties, ClientProperties.java, ServerProperties.java, or setup.sh. Make sure that all of these files have up to date information and that the jars have been packaged once all information is correct.



I am receiving connection refused errors when running FileSyncClient.jar

You have either not allowed port 5000 on your server or FileSyncHead.jar is not running on the server. Please check that you have both and try again.



There's already a process running on port 5000?

Using the "top" command on your server you can see what processes are running. There should be a java process that pops up every once in a while. Note the process id. Type "kill" and then the process id to kill this process. You should now be able to run FileSyncHead.jar again.



The download links are not working for the files on my home page.

Unfortunately, the download links will only remain active as long as you keep the instance on which they are stored running. We did not provide a way to delete files from the MySql table when an instance in shut down so those old file links will remain on your webpage. If you start a new instance with the same folder, the files should be uploaded onto the new instance and the download link should point to those most recent files.
